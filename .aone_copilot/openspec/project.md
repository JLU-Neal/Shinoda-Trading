# 项目上下文

## Purpose
Shinoda Trading 是 Shinoda Trading 公司的首个量化交易系统，旨在构建一套自动化的量化交易策略执行平台，支持策略研发、回测、模拟交易和实盘交易等核心功能。

## 技术栈
- **编程语言**: Rust
- **构建工具**: Cargo（Rust 官方包管理器和构建系统）
- **IDE**: RustRover / IntelliJ IDEA（JetBrains 系列）
- **代码格式化**: rustfmt
- **代码检查**: clippy
- **变异测试**: cargo-mutants

## 项目约定

### 代码风格
- 遵循 Rust 官方编码规范（Rust API Guidelines）
- 使用 `rustfmt` 进行代码格式化，保持一致的代码风格
- 使用 `clippy` 进行静态代码分析，确保代码质量
- 命名约定：遵循 Rust 标准的 snake_case（变量/函数）和 CamelCase（类型/trait）命名规范
- 优先使用 Rust 的所有权和借用机制，避免不必要的 clone
- 错误处理优先使用 Result 和 Option 类型，避免 unwrap/panic

### 架构模式
- 项目处于初始阶段，架构待定
- 预期采用模块化设计，将策略引擎、数据获取、订单管理、风控等功能拆分为独立模块
- 追求高性能和低延迟，充分利用 Rust 的零成本抽象和内存安全特性

### 测试策略
- 使用 Rust 内置的测试框架（`#[cfg(test)]` 和 `#[test]`）
- 单元测试与源代码放在同一文件中
- 集成测试放在 `tests/` 目录下
- 使用 cargo-mutants 进行变异测试以评估测试质量

### Git 工作流
- 主分支为 main
- 功能开发使用 feature 分支
- 提交信息应清晰描述变更内容

## 领域上下文
- **量化交易**: 使用数学模型和算法进行金融市场交易决策
- **核心概念**: 交易策略、回测引擎、订单管理、风险控制、市场数据、K线/Tick数据、技术指标
- **性能要求**: 量化交易系统对延迟极为敏感，需要高吞吐量和低延迟的数据处理能力
- **数据处理**: 需要处理大量的历史和实时市场数据

## 重要约束
- 交易系统涉及资金安全，代码必须经过严格测试和审查
- 需要确保系统的高可用性和容错能力
- 遵守相关金融市场法规和合规要求
- Rust 的内存安全特性有助于避免常见的安全漏洞

## 外部依赖
- 项目处于初始阶段，外部依赖待确定
- 预期可能涉及：市场数据 API、交易所接口、数据库存储等
