## Context
当前回测系统支持两种策略：动量策略和简单均值回归策略。用户需要一种基于统计偏离度的更精确均值回归策略，使用 z-score 来量化价格偏离均值的程度。

## Goals / Non-Goals
- **Goals**:
  - 实现基于 z-score 的均值回归策略，使用滚动窗口内的总体标准差
  - 支持参数校验，非法参数在构造时即失败
  - 通过 CLI 参数 `--entry-z` 和 `--exit-z` 控制策略阈值
- **Non-Goals**:
  - 不实现做空逻辑
  - 不修改现有策略的行为
  - 不引入新的外部依赖

## Decisions

### 使用总体标准差
- **决策**: 使用总体标准差 `σ = sqrt(sum((x_i - μ)^2) / N)` 而非样本标准差
- **理由**: 窗口内的数据即为总体，不存在从样本推断总体的需求；与用户规范一致
- **替代方案**: 样本标准差 `σ = sqrt(sum((x_i - μ)^2) / (N-1))` → 语义上不如总体标准差准确

### 构造函数返回 Result
- **决策**: `new()` 返回 `Result<Self, String>`，参数非法时返回错误
- **理由**: 遵循用户规范要求，参数非法应在构造时即失败，而非运行期隐式容忍；与现有策略的 `new()` 直接返回 `Self` 不同，但更安全
- **替代方案**: 直接 panic → 不符合 Rust 错误处理最佳实践

### 信号规则中的阈值方向
- **决策**: Buy 条件为 `z_t < -entry_z`，Sell 条件为 `z_t > -exit_z`
- **理由**: 遵循用户规范定义的 long-only 逻辑；负 z-score 表示价格低于均值，当偏离足够大时买入，回归时卖出

## Risks / Trade-offs
- **与现有策略接口差异**: 新策略的 `new()` 返回 `Result`，而现有策略直接返回 `Self` → `build_strategy` 中需要处理 `Result`，但不影响现有策略
- **CLI 参数命名**: `--entry-z` 和 `--exit-z` 与现有的 `--entry-threshold`/`--exit-threshold` 命名风格一致
